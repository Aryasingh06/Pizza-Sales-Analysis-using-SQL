Pizza Sales Analysis Using SQL

# Project Overview
This project analyzes a pizza sales dataset using MySQL to uncover valuable business insights.
The analysis focuses on customer ordering patterns, revenue generation, product performance, category-wise sales,
and time-based trends using SQL queries.
The project demonstrates practical SQL skills including database creation, table relationships, 
joins, aggregate functions, subqueries, window functions, and business analytics.

## Project Objectives
- Design a relational database for pizza sales data.
- Import and manage data from multiple CSV files.
- Perform SQL-based business analysis.
- Generate insights to support business decision-making.
- Showcase SQL skills through real-world business scenarios.

 ## Dataset Description
The project uses four related datasets:

Tables                  Description
orders            Stores customer order date and time.
order_details     Stores pizzas ordered and quantity.
pizzas            Contains pizza size and price information.
pizza_types       Contains pizza name, category, and ingredients.

## Business Questions Solved

Order Analysis
- Total number of orders placed.
- Most commonly ordered pizza size.

Revenue Analysis
- Total revenue generated.
- Highest-priced pizza.
- Revenue contribution by pizza category.
- Cumulative revenue over time.

Product Analysis
- Highest-priced pizza.
- Top 5 most ordered pizza types.

Category Analysis
- Total quantity ordered by category.
- Number of pizza types in each category.
- Top 3 pizza types based on revenue.

Time-Based Analysis
- Average pizzas ordered per day.
- Distribution of orders by hour.

Additional Business Analysis
- Pizza size generating the highest revenue.
- Revenue by weekday.
- Revenue by month.
- Average revenue per order.
- Bottom 5 revenue-generating pizzas.

 ## Key Insights
- Identified the overall revenue generated by pizza sales.
- Determined customer ordering trends based on time.
- Identified the most popular pizza sizes and categories.
- Measured category-wise contribution to revenue.
- Analyzed cumulative revenue growth over time.
- Evaluated product performance using revenue and order quantity.

## How to Run This Project
1. Create the database using "schema.sql".
2. Import all four CSV files into their respective tables.
3. Execute the queries in "analysis_queries.sql".
4. Review the query outputs and business insights.
